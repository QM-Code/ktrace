# Foundation Folder Guidance

`docs/foundation/` is the canonical home for long-lived documentation.

Subdirectory guidance:
- `docs/foundation/policy/AGENTS.md`
- `docs/foundation/governance/AGENTS.md`
- `docs/foundation/architecture/AGENTS.md`

Rules:
- Do not keep transient project status or assignment state here.
- Do not duplicate foundation rules in `docs/projects/`.
- Treat `docs/projects/` as transient execution plans that can be archived without losing durable knowledge.
